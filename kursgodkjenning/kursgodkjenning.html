MET4 – Obligatorisk kursgodkjenning

Dataanalyse med R, basert på modul 1–4

ViktigRegler for kursgodkjenningen
  • Dette er en individuell obligatorisk oppgave. Du skal løse den selv.
  • Oppgavene løses i R-vinduene under. R kjører rett i nettleseren, så du trenger ikke installere noe.
  • Hver oppgave rettes automatisk: skriv koden din, trykk Run, og du får umiddelbart vite om svaret er riktig.
  • Det er 16 deloppgaver fordelt på fire hovedoppgaver. Du kan bruke nettsiden, notatene dine og hjelpesidene i R fritt.
  • Når alle oppgavene er grønne, fyller du inn kandidatnummeret ditt nederst og får en fullføringskode. Last den opp i Canvas innen fristen. (Frist: se Canvas.)
NoteSlik bruker du kodevinduene

Første gang du kjører en kodecelle, laster nettleseren ned R. Det tar noen sekunder, men skjer bare én gang. Skriv koden din, trykk Run (eller Shift+Enter), så rettes svaret. Du kan prøve så mange ganger du vil, og trykke Show hints hvis du står fast.

To typer oppgaver går igjen:

  • Regneoppgaver. Sørg for at den siste linjen i cellen regner ut selve svaret, altså tallet. Skriver du t.test(...) får du hele utskriften; skal du levere p-verdien, må siste linje være t.test(...)$p.value. Ender du opp med en tabell på én rad og én kolonne, slik summarise() gir deg, godtas den også.
  • Tolkningsoppgaver. Her svarer du med bokstaven til det alternativet du mener er riktig, som en tekststreng: skriv "b" på siste linje. Du får en forklaring uansett hva du svarer.

Noen celler er merket Utforsk. De rettes ikke og teller ikke med i fremdriften; de er der for at du skal lage figuren eller utskriften du trenger for å svare på tolkningsoppgaven rett etter.

dplyr og ggplot2 er ferdig lastet i alle cellene, så du kan bruke %>%, filter(), group_by(), summarise() og ggplot() som du gjorde i modul 1. Base R fungerer like godt; rettingen bryr seg bare om svaret.

Dataene

Du skal analysere annonser for utleieboliger i Bergen. Et eiendomsselskap vurderer om det lønner seg å møblere boligene de leier ut, og har hentet inn data om 900 utleieannonser.

Datasettet boliger er allerede lastet inn i hver oppgavecelle, sammen med pakkene du trenger:

library(dplyr)
library(ggplot2)
load("boliger.RData")

Filen inneholder også datasettet nye, som er fem boliger selskapet skal legge ut for utleie, og som du bruker i oppgave 3d.

Oppgave 1 – Deskriptiv statistikk

Oppgave 1a

Hvor mange av boligene i utvalget ligger i Bergenhus og leies ut møblert? La siste linje gi antallet.

Med filter() fra modul 1 kan du sette flere kriterier samtidig ved å skille dem med komma, og n() inne i summarise() teller radene som er igjen. Vil du heller bruke base R, gir boliger$bydel == "Bergenhus" en vektor av TRUE/FALSE, to slike kan kombineres med &, og sum() teller antallet TRUE. En tredje vei er table(boliger$bydel, boliger$mobelert), som gir hele krysstabellen på én gang.

boliger %>%
  filter(bydel == "Bergenhus", mobelert == "Ja") %>%
  summarise(antall = n())

Eller i base R:

sum(boliger$bydel == "Bergenhus" & boliger$mobelert == "Ja")

Oppgave 1b

Regn ut korrelasjonen mellom pris og areal.

cor(x, y) regner ut korrelasjonen mellom to vektorer.

cor(boliger$pris, boliger$areal)

Oppgave 1c

Kjør cellen under for å lage to boksplott: leiepris per bydel, og areal per bydel.

Utforsk (denne cellen rettes ikke):

Hvilken påstand er best i tråd med det de to figurene viser?

a) Bergenhus har de høyeste leieprisene, noe som er forventet siden bydelen ligger mest sentralt.

b) Fana og Åsane har høyere medianpris enn Bergenhus og Årstad, og en nærliggende forklaring er at boligene der er større.

c) Bydel har ingen betydning for hva en bolig koster.

d) Prisforskjellene mellom bydelene er så store at areal ikke kan ha noe å si for prisen.

Sammenlign de to figurene. Bydelen med høyest medianpris: er den også bydelen med de største boligene?

Oppgave 1d

Regn ut medianprisen i hver bydel, og la svaret bli differansen mellom den høyeste og den laveste av de fire medianene.

Dette er group_by() og summarise() fra modul 1: del dataene inn i grupper, og regn ut ett tall per gruppe. Du står igjen med fire medianer, og på dem kan du bruke max() og min(), enten i en ny summarise() eller direkte.

boliger %>%
  group_by(bydel) %>%
  summarise(median_pris = median(pris)) %>%
  summarise(spenn = max(median_pris) - min(median_pris))

Eller i base R:

medianer <- tapply(boliger$pris, boliger$bydel, median)
max(medianer) - min(medianer)

Oppgave 2 – Hypotesetesting

Selskapet vil vite om møblerte boliger leies ut til en annen pris enn umøblerte. I denne oppgaven sammenligner vi de to gruppene direkte.

Oppgave 2a

Før vi sammenligner gjennomsnittene, må vi vite om vi kan anta lik varians i de to gruppene. Utfør en F-test for like varianser i pris mellom møblerte og umøblerte boliger, og la svaret bli p-verdien.

var.test() tester \(H_0: \sigma_1^2 = \sigma_2^2\). Formelsyntaksen pris ~ mobelert betyr «pris gruppert etter møblering», og fungerer på samme måte som i t.test().

Kjører du bare var.test(pris ~ mobelert, data = boliger), får du hele utskriften, men oppgaven ber om ett tall. Lagre testen i en variabel og hent ut komponenten du vil ha med $:

test <- var.test(pris ~ mobelert, data = boliger)
test$p.value

Du kan også hekte $p.value rett på funksjonskallet. Samme framgangsmåte gjelder for t.test() og chisq.test() senere i settet, der du trenger $statistic.

test <- var.test(pris ~ mobelert, data = boliger)
test$p.value

Eller i én linje:

var.test(pris ~ mobelert, data = boliger)$p.value

Oppgave 2b

Hva følger av resultatet i 2a for valget av t-test?

a) Variansene er signifikant forskjellige, så vi må bruke Welch-testen, altså var.equal = FALSE.

b) Vi forkaster ikke \(H_0\) om like varianser, så vi kan bruke den poolede t-testen, altså var.equal = TRUE.

c) Den høye p-verdien viser at variansene er nøyaktig like i de to gruppene.

d) F-testen sier ingenting om varianser, bare om gjennomsnitt, så den er irrelevant her.

Nullhypotesen i var.test() er at de to variansene er like. Hva gjør du med en nullhypotese når p-verdien er 0,29, og hva slags t-test tillater det deg å bruke?

Oppgave 2c

Utfør en tosidig to-utvalgs t-test av om gjennomsnittlig pris er forskjellig for møblerte og umøblerte boliger. Bruk varianten du kom fram til i 2b. La svaret bli testobservatoren \(t\).

t.test(pris ~ mobelert, data = boliger, var.equal = TRUE) utfører testen. Kjører du den uten mer, får du hele utskriften, og der står testobservatoren på linjen som begynner med t =. Du kan lese tallet derfra og skrive det inn selv.

Vil du heller hente det ut programmatisk, slik du gjorde med p-verdien i 2a, er komponenten $statistic:

test <- t.test(pris ~ mobelert, data = boliger, var.equal = TRUE)
test$statistic

Fortegnet avhenger av hvilken gruppe R setter først, og spiller ingen rolle her; begge fortegn godtas.

t.test(pris ~ mobelert, data = boliger, var.equal = TRUE)$statistic

Oppgave 2d

Testen i 2c var tosidig: den spør bare om det finnes en forskjell, uten å ta stilling til retningen. Men selskapet har en klar formening. De tror at møblering gir høyere leie, og det er den eneste retningen som er relevant for beslutningen deres: er premien null eller negativ, møblerer de ikke.

Gjennomfør derfor testen på nytt som en ensidig test av selskapets hypotese:

\[H_0: \mu_{\text{møblert}} = \mu_{\text{umøblert}} \qquad \text{mot} \qquad H_A: \mu_{\text{møblert}} > \mu_{\text{umøblert}}\]

La svaret bli p-verdien. Vær nøye med retningen: mobelert har nivåene "Nei" og "Ja" i den rekkefølgen.

Du styrer retningen med alternative-argumentet, som kan være "less" eller "greater".

Nøkkelen er hvilken vei R regner differansen. Med formelsyntaks bestemmes det av nivårekkefølgen i den kategoriske variabelen, og mobelert har "Nei" før "Ja". R ser altså på Nei - Ja. Skal du teste at de møblerte er dyrere, må Nei - Ja være negativ.

Sjekk alltid utskriften: t.test() skriver eksplisitt ut hva den tester, for eksempel "alternative hypothesis: true difference in means between group Nei and group Ja is less than 0".

test <- t.test(pris ~ mobelert, data = boliger,
               var.equal = TRUE, alternative = "less")
test$p.value

Oppgave 2e

Testen i 2c viser at møblerte boliger i utvalget har klart lavere leie enn umøblerte. Hvilken konklusjon er den riktige?

a) Testen viser at det å møblere en bolig gjør den billigere å leie ut.

b) Møblerte boliger i utvalget har lavere leie, men testen sier ikke at møbleringen er årsaken. Møblerte og umøblerte boliger kan skille seg fra hverandre på andre måter som også påvirker prisen.

c) Fordi p-verdien er ekstremt liten, er forskjellen på 2 353 kroner også økonomisk stor.

d) Testen er ikke gyldig, siden de to gruppene ikke er like store.

Se tilbake på oppgave 1a og 1c. Hvor ligger de møblerte boligene, og hvor store er boligene der?

Oppgave 2f

Selskapet lurer også på om møblering er like vanlig i alle bydeler. Lag en krysstabell over bydel og mobelert, og utfør en kjikvadrattest for uavhengighet. La svaret bli testobservatoren \(\chi^2\).

chisq.test(table(boliger$bydel, boliger$mobelert)) gir hele testen. Testobservatoren hentes ut med $statistic.

chisq.test(table(boliger$bydel, boliger$mobelert))$statistic

Oppgave 3 – Regresjon

Nå skal vi forsøke å isolere hva møbleringen faktisk er verdt.

Oppgave 3a

Tilpass en enkel lineær regresjon med pris som responsvariabel og mobelert som eneste forklaringsvariabel. La svaret bli koeffisienten for møblering.

lm(pris ~ mobelert, data = boliger) tilpasser modellen. coef() gir deg koeffisientene, og du plukker ut nummer to med [2]. Siden mobelert har nivåene "Nei" og "Ja", er "Nei" referansen, og koeffisienten heter mobelertJa.

modell <- lm(pris ~ mobelert, data = boliger)
coef(modell)[2]

Oppgave 3b

Utvid modellen slik at den kontrollerer for boligens egenskaper. Tilpass

pris ~ mobelert + areal + soverom + bydel + byggeaar + balkong

og la svaret bli koeffisienten for møblering i denne modellen.

Formelen skrives med + mellom forklaringsvariablene. coef(modell)["mobelertJa"] henter ut akkurat den koeffisienten du er ute etter, uansett hvilken plass den har i rekken.

modell <- lm(pris ~ mobelert + areal + soverom + bydel + byggeaar + balkong, data = boliger)
coef(modell)["mobelertJa"]

Oppgave 3c

I 3a var koeffisienten for møblering -2 353. I 3b er den +1 905. Hva er den beste forklaringen på at fortegnet snudde?

a) Fortegnet snudde fordi den multiple modellen bruker flere observasjoner enn den enkle.

b) Møblerte boliger er systematisk mindre enn umøblerte, i snitt 51 mot 77 kvadratmeter. I den enkle modellen måler møblert-koeffisienten både møbleringen og størrelsesforskjellen, og størrelsen dominerer. Når areal holdes fast, isolerer koeffisienten selve møbleringen.

c) Fortegnet snudde fordi areal og møblering er ukorrelerte, slik at de to modellene måler helt ulike ting.

d) Den enkle modellen er den riktige. Den multiple modellen er overtilpasset og bør forkastes.

Kjør boliger %>% group_by(mobelert) %>% summarise(snitt_areal = mean(areal)) i en av cellene over. Hvor store er de møblerte boligene sammenlignet med de umøblerte, og hvordan påvirker det en sammenligning som ikke tar hensyn til areal?

Oppgave 3d

Selskapet skal legge ut fem nye boliger for utleie. De ligger i datasettet nye. Bruk modellen fra 3b til å predikere leieprisen for alle fem, og la svaret bli den høyeste predikerte prisen.

predict(modell, newdata = nye) gir en vektor med fem prediksjoner, én per bolig. max() plukker ut den største. Skriv nye i en celle først hvis du vil se hvilke boliger det er snakk om.

modell <- lm(pris ~ mobelert + areal + soverom + bydel + byggeaar + balkong, data = boliger)
max(predict(modell, newdata = nye))

Oppgave 4 – Utfordring

Denne siste oppgaven er tøffere, og bygger på alt det foregående.

Oppgave 4a

Selskapet mistenker at møblering er verdt mer i sentrum, der studenter og pendlere flytter oftere, enn i bydelene lenger ut. For å teste det trenger vi en modell der møbleringseffekten får lov til å variere mellom bydelene. Det får vi med et interaksjonsledd, som skrives mobelert * bydel:

pris ~ mobelert * bydel + areal + soverom + byggeaar + balkong

Tilpass modellen, og la svaret bli koeffisienten mobelertJa. Siden "Bergenhus" er referansenivået for bydel, er dette møbleringspremien i Bergenhus.

mobelert * bydel er kortform for mobelert + bydel + mobelert:bydel, altså begge hovedeffektene pluss samspillet. Du får hele bildet med summary(modell), og henter ut den ene koeffisienten med coef(modell)["mobelertJa"].

modell <- lm(pris ~ mobelert * bydel + areal + soverom + byggeaar + balkong, data = boliger)
coef(modell)["mobelertJa"]

Oppgave 4b

Kjør summary() på modellen fra 4a i cellen under, og se på de tre interaksjonsleddene nederst i koeffisienttabellen.

Utforsk (denne cellen rettes ikke):

Hvilken påstand er riktig?

a) Møbleringspremien er den samme i alle fire bydelene.

b) Møbleringspremien er størst i Bergenhus, rundt 3 000 kroner, og lavere i de tre andre bydelene. Alle tre interaksjonsleddene er signifikante på 5 prosent nivå.

c) Interaksjonsleddene viser at areal påvirker prisen ulikt fra bydel til bydel.

d) Siden interaksjonsleddene er negative, er møblering verdiløst utenfor Bergenhus.

Interaksjonsleddene er de tre radene med kolon i navnet, for eksempel mobelertJa:bydelÅrstad. Møbleringspremien i Årstad er mobelertJa pluss mobelertJa:bydelÅrstad.

Fullføring

Oversikten under oppdateres automatisk etter hvert som oppgavene blir grønne, så du ser hvilke som gjenstår. Når alle 16 er løst, skriver du inn kandidatnummeret ditt, så dukker fullføringskoden opp.

Merk at fremdriften ligger i nettleserens minne. Laster du siden på nytt, nullstilles den, og oppgavene må kjøres om igjen. Det tar bare noen sekunder når koden din allerede står i cellene, men det kan være greit å ta kursgodkjenningen i én økt.

// Leser av hvilke oppgaver som er løst. Oppgavelista utledes fra sidens egen
// DOM, i dokumentrekkefølge, slik at den holder seg i takt om oppgaver legges
// til eller fjernes. Generatoren yielder bare når noe faktisk har endret seg,
// så visningen under ikke tegnes på nytt hvert halvsekund.
oppgaveSignatur = {
  let forrige = null;
  while (true) {
    const alle = [];
    for (const el of document.querySelectorAll("[data-exercise]")) {
      const id = el.getAttribute("data-exercise");
      if (id && !alle.includes(id)) alle.push(id);
    }
    const loste = [];
    for (const g of document.querySelectorAll(".exercise-grade.alert-success")) {
      const wrap = g.closest("[data-exercise]");
      const id = wrap && wrap.getAttribute("data-exercise");
      if (id && !loste.includes(id)) loste.push(id);
    }
    const signatur = JSON.stringify({ alle, loste });
    if (signatur !== forrige) {
      forrige = signatur;
      yield signatur;
    }
    await new Promise((r) => setTimeout(r, 400));
  }
}
status = JSON.parse(oppgaveSignatur)
antallRiktige = status.loste.length
totalOppgaver = 16
fremdrift = {
  const ferdig = antallRiktige >= totalOppgaver;
  const farge = ferdig ? "#22aa66" : "#666";
  const lost = new Set(status.loste);

  // Grupper etter hovedoppgave: ex2c -> "2"
  const grupper = new Map();
  for (const id of status.alle) {
    const merke = id.replace(/^ex/, "");
    const hoved = merke.charAt(0);
    if (!grupper.has(hoved)) grupper.set(hoved, []);
    grupper.get(hoved).push({ merke, ok: lost.has(id) });
  }

  const brikke = (b) => html`<span
    title=${b.ok ? "Oppgave " + b.merke + " er riktig" : "Oppgave " + b.merke + " gjenstår"}
    style="display:inline-flex;align-items:center;gap:0.3em;
           padding:0.2em 0.6em;border-radius:999px;font-size:0.95em;
           font-variant-numeric:tabular-nums;
           border:1px solid ${b.ok ? "#22aa66" : "#ccc"};
           background:${b.ok ? "#eafbf1" : "transparent"};
           color:${b.ok ? "#177a49" : "#888"}">
    ${b.ok ? "✓" : "○"} ${b.merke}
  </span>`;

  const rader = [...grupper.entries()].map(

                                           gap:0.5em;margin:0.35em 0;flex-wrap:wrap">
      <span style="color:#888;font-size:0.9em;min-width:6.5em">Oppgave ${hoved}</span>
      ${brikker.map(brikke)}
    </div>`
  );

  return html`<div style="margin:0.8em 0">
    <div style="font-size:1.15em;margin-bottom:0.5em">

            .map((id) => id.replace(/^ex/, ""))
            .join(", ")}
        </div>`}
  </div>`;
}
